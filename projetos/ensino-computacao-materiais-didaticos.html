Estudo de Caso 1

Trilha de lógica de programação

Estudo para estruturar aprendizagem prática, acompanhar evolução e criar materiais claros.

Overview

Estudo aplicado ao contexto de ensino de computação e materiais didáticos, com foco em organizar dados, processos e indicadores para análise recorrente.

Problema

Conteúdos e atividades precisavam ser organizados para facilitar prática, revisão e acompanhamento.

  • Diferenças de ritmo entre participantes.
  • Materiais dispersos.
  • Dificuldade de medir evolução por tópico.

Objetivos

  • Criar sequência didática.
  • Aumentar prática orientada.
  • Acompanhar evolução.
  • Facilitar revisão e autonomia.

Dataset

Fonte principalAtividades, formulários e planilhas.
Fonte auxiliarMateriais, guias, slides e exemplos.
ControlePlanilhas, CSVs ou registros operacionais de apoio.

ETL

  1. Extração das fontes.
  2. Limpeza e padronização.
  3. Modelagem da base analítica.
  4. Criação de KPIs.
  5. Publicação para análise.

Diagnóstico → Aula → Prática → Feedback

Imagem do dashboardTrilha de lógica de programação

SQL

SELECT
    categoria,
    COUNT(*) AS registros,
    SUM(valor) AS total
FROM aprendizagem
GROUP BY categoria
ORDER BY total DESC;

Insights

  • Exemplos práticos aumentaram participação.
  • Projetos pequenos aumentaram engajamento.
  • Dados ajudaram a ajustar ritmo.

Resultado

A organização dos materiais tornou a aprendizagem mais prática e mensurável.
